CSS 最全的总结和解决CSS问题的办法

1.CSS选择器的种类

1.通用选择器 eg:
2.id选择器 eg: #X
3.类选择器 eg:.X
4.元素选择器 eg: X
5.后代选择器 eg: X Y
6.相邻选择器 :在X之后第一个兄弟节点中选择满足Y选择器的元素 eg: X+Y
7.后代选择器 :选择X的子元素中满足Y选择器的元素 eg: X>Y
8.兄弟选择器:选择X之后所有的兄弟节点满足Y选择器的元素 eg: X~Y
9.属性选择器 eg: [title] {color:red;}
属性选择器包括：
a.[attr]:选择所有设置了attr属性的元素，兼容性：IE7+
b.[attr=value]:attr选择属性值刚好为value的元素
c.[attr~=value]:attr选择属性值其中一个的值刚好是value或者有空格分开的value值的元素
d.[attr|=value]:attr选择属性值刚好为value或者value-开头的元素
e.[attr^=value]:attr选择属性值以value开头的元素
f.[attr$=value]:attr选择属性值以value结尾的元素
g.[attr*=value]:attr选择属性值中包含value的元素
h.input:checked :选择单选框，复选框，下拉框中选中状态下的元素
10.伪类选择器 eg: a:hover , li:nth-child

2.CSS选择器的优先级

!important > 内联 > id选择器 > 类选择器 > 标签选择器。其中，!important 用于强调CSS属性具有最高的优先级。多个类选择器叠加（256）之后的优先级大于一个id选择器。
CSS优先级的计算规则如下：
元素标签中定义的样式（Style属性）,加1,0,0,0
每个ID选择符(如 #id),加0,1,0,0
每个Class选择符、每个属性选择符、每个伪类（:hover）加0,0,1,0
每个元素选择符（如p）或伪元素选择符(如 :firstchild)等，加0,0,0,1
然后，将这四个数字分别累加，就得到每个CSS定义的优先级的值
然后从左到右逐位比较大小，数字大的CSS样式的优先级就高。

3.超链接访问过后hover样式就不出现的问题

原因：CSS属性的排列顺序不正确，会导致被点击访问过的超链接样式不在具有hover和active了,解决方法是改变CSS属性的排列顺序:L-V-H-A（即先设置link ,再设置visited,接着hover,最后是active）

4.CSS hack

由于不同的浏览器，比如Internet Explorer 6,Internet Explorer 7,Mozilla Firefox等，对CSS的解析认识不一样，因此会导致生成的页面效果不一样，得不到我们所需要的页面效果,这就会产生CSS hack的问题.解决的办法： 我们就需要针对不同的浏览器去写不同的CSS，让它能够同时兼容不同的浏览器，能在不同的浏览器中也能得到我们想要的页面效果。

5.display:none与visibility:hidden的区别是什么？

display : 隐藏对应的元素但不挤占该元素原来的空间 ，但是使用CSS display:none属性后，HTML元素（对象）的宽度、高度等各种属性值将会“丢失”。
visibility: 隐藏对应的元素并且挤占该元素原来的空间，使用visibility:hidden属性后，HTML元素（对象）仅仅是在视觉上看不见（完全透明），它依然占据的空间位置。

6.CSS的盒子模型

CSS盒子模型从内到外分别是 元素的内容（element content）,内边距（padding）,边框（border）,外边距（margin）。

如图所示，元素的高度和宽度指的是盒子模型的content区域的高度和宽度，不包括内边距以外的部分。(IE的content则是包含border、内边距和content)。盒子模型有两种，W3C标准盒子模型（content-box）以及IE盒子模型（border-box）。中间蓝色部分表示 元素的内容，外面绿色部分表示内边距，黄色部分表示边框，最外层橙色部分表示外边距。

6.相对定位relative、浮动float以及绝对定位absolute的区别？

相对定位(relative)：相对于自身的定位。
注意，在使用相对定位时，无论是否进行移动，元素仍然占据原来的空间。因此，移动元素会导致它覆盖其它框。
绝对定位(absolute)：相对于其他元素的定位。
注意：绝对定位使元素的位置与文档流无关，因此不占据空间。它是漂浮在页面中。
浮动框就是一个框在当前行被向左或向右挪动(偏移)，它不在常规流中。浮动后的块元素相当于一个inline-block元素。

7.清除浮动

浮动的元素会产生一些不好的结果，使得内容溢出到容器外面而影响（甚至破坏）布局的现象。因此要清除浮动。以下有几种方法清浮动：
1.在浮动元素加入class=”clear”，并在CSS中赋予.clear{clear:both;}属性即可清理浮动。
2.给浮动元素的容器添加overflow:hidden;或overflow:auto;可以清除浮动
3.给直接在浮动元素后面的元素添加clear属性 eg:clear:both;
4.使用CSS的:after伪元素清除浮动。利用伪类清除浮动，给浮动元素的容器添加一个clearfix的class，然 后给这个class添加一个:after伪元素实现元素末尾添加一个看不见的块元素（Block element）清理浮动。

8.BOX模型的margin在IE比在其他浏览器大2px的解决办法

在IE浏览器中Box模型的margin比其他浏览器的Box模型大2px，所以解决办法有：div{margin:30px!important;margin:28px;}.
